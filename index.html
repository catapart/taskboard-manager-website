About

Keep it simple

The <taskboard-manager> custom element is an app that provides a way to track tasks in a similar way to Kanban boards.

It's free to use, it's private, and it works in desktop and mobile screen sizes, in both light and dark modes.

Use the app to manage projects, keep up with reminders, shop for groceries, and more! No login or internet connection required.

Back up

Not sure what a custom element even is? That's okay! It's just a specific kind of package for developers to use. You don't need to know anything about it, in order to use the task manager app.

Unfamiliar with "Kanban" or task management? No worries! You can think of the app as a complicated "To Do" list editor. You can create lists of tasks and then indicate when those tasks have been completed.

That should give you a general idea of what you could or couldn't use the app for. For a little more background, though...

Custom Elements

A custom element is a special type of javascript class that can be represented in the HTML programing language as markup. That's a fancy way of saying that if you have a .html file, you can write <taskboard-manager></taskboard-manager> in it, and that will make the entire app appear in your webpage. There are caveats to that - like having to include the javascript that drives the app - but it's pretty much that easy.

In fact, if you right-click (or long press) on this page and choose the option to "View Page Source", you can see that the element is in the header of this page (which is how that header is interactive).

If you are a developer, you may have heard of custom elements by the psuedonym "Web Components". That's mostly how I have heard them talked about. But it's not in any spec that I've seen, so I hesitate to use it in documentation. It seems to mostly be a way to demystify the concept for people familiar with javascript frameworks. Custom Elements are to javascript as Components are to React, for example. So that's fine, by me!

In that sense, the <taskboard-manager> is a web component. And it's actually made up of web components! You can find out more in the Custom Elements section.

"Kanban" style boards

A "kanban" board is a collection of lists that each contain tasks. The board manages everything together so that tasks from one list can be moved to other lists.

These boards, and their lists can be used to keep track of all kinds of things, from grocery lists to project management. The ability to assign tasks to any list on the board allows you to create "lanes" that can describe where a task is on its way to being "done". That creates a unique flexibility for task-tracking.

It may be hard to imagine why you might "need" multiple lanes for a grocery list, but you may not! Since each task has its own checkbox, you can use a single list as a grocery list and never worry about moving tasks around.

On the other hand, if you are planning a trip, you might need to make many lists for a single trip. One list for what to pack, another for food, another for travel and lodging, etc. It's nice to be able to keep all of them together on a single board.

Where the board shines, though, is in project management. Each board is initialized with lanes that correspond to typical (if simplified) project management task tracking. Instead of just tracking what tasks need to be done, you can easily track how far along a task is in being done. This type of modularity and granularity in a project can sharpen focus and keep the project moving at a fast pace!

If you would like some ideas for getting started, check out the board templates section.

(I keep putting "kanban" in quotes because I think it's a formal concept which has a lot more to it than just "fancy to-do lists", and I don't want to imply I'm actually representing what kanban boards are. For more information about acutal kanban, check out the wikipedia article!)

Take note

If you are really interested in using the app for task management, there are a few things to be aware of. Firstly, the data is stored in a place on your computer that may get deleted. If you are going to use the app, please make sure to export your content frequently. Files that are exported from the app are saved to your device and will not be deleted automatically.

Next, if you would like to use the app offline, without needing to connect to the internet, you will need to install it. Follow this guide to install the app onto your device (desktop, laptop, or mobile). Once you've installed the app, you can always use the app - even without an internet connection!

Also, the data is local to your device. That means that - unlike other web apps - you will not share data between your phone and your desktop computer. This is because - unlike other web apps - this website's servers do not store you data. Your device stores your data. So the only way to get it from one device to another is by using the export and import process.

Another thing to know is that the website does not guarantee uptime at all. If you are using the app without installing it by visiting the website, then you are risking being locked out of the app if the site goes down. It's not my intention to let the site go down, but things happen and - in my experience - they tend to happen at the worst possible times. More than anything, this site is a showcase and a reference for development of web apps using these technologies. So please do not rely on it to provide access to your data. Again, you can install the app to always have access to it. And even without the app, there are ways to retrieve the data stored in it, so you aren't at risk of losing your data if the site goes down. This is mainly just a disclaimer for possible outages.

    Icons

    Since some of the custom elements use icons, and the elements were intended to be released in the public domain, I needed a set of icons that were also in the public domain. Rather than find a set, I decided to make my own, as I needed them.

    While the set is not very comprehensive, it is nonetheless completely free for anyone to use! They were each designed and developed by hand in Inkscape, with an intention of using them in these custom elements. That means that each one biases toward simple shapes and readability at small scales. They are also all optimized for size, since the intention was to inject each one into HTML. So there are no groups with odd translations, or unused app ids, or whatever other junk is often injected into SVG content. In fact, if you are familiar with HTML, you can open the icons in a text editor and probably be able to understand what each element is used for.

    Design

    Familiar

    One of the goals of the <taskboard-manager> and all of the custom elements that make it up was to mimic the developer experience for standard HTML elements.

    In the same way that you might still need to look up the exact api for a "file"-type <input>, or how to utilize a <datalist>, or whatever, you will still need to refer to documentation to use these elements.

    That aside, if you are familiar with web technologies, most of what these custom elements do and expect should be familiar to you. For example, events are dispatched and bubble from <task-card> right on up through each parent element in the same way that "click" events do.

    Other examples would be that the <editable-list> element implements the same margins and padding that <ul> and <ol> elements do, or that all of the elements defer to dispatching change events, when it is appropriate, to match that functionality from input elements.

    In addition to feeling familiar, the custom elements also rely on standard HTML capabilties wherever possible. This includes buttons and links for accessibility where applicable, as well as leveraging css and inputs in place of javascript when possible.

    Where custom solutions had to be used, these custom elements have tried to implement common-sense features. This includes things like css variables for border colors, or slots specifically to replace icons.
